# M8 实施计划 — 共享 Agent 会话与反向 MCP

## 目标与验收

M8 把 Rocket.Chat 话题作为共享 Agent 会话：消息始终先进入 Rocket.Chat，宿主客户端只在 `@codex` 或 `$` 指令出现时调用本机 Codex；完整回复回到话题，过程事件留在本地 Agent 面板。

完成必须同时满足：

- `codex app-server` 协议版本锁定，当前基线为 `codex-cli 0.144.4`；生成类型可重复生成并验证无漂移。
- 当前版本全部 server-initiated request 都有明确响应路径，未知请求安全拒绝。
- 会话单写者、指令串行、主持人租约、非主持人首次放行、执行/写入仅主持人审批成立。
- 默认只读且受工作区白名单和敏感路径拒绝约束；回帖前执行密钥样式脱敏。
- agent 进程被杀后会话进入“已中断”，重新启动后可 resume。
- 官方 Rocket.Chat 客户端能看到完整话题；本地 Agent 面板能看到工具和审批过程。
- `rcx-mcp` 能让外部 Agent 在明确账号和会话范围内读取聊天上下文。

## 已确认事实

- `codex app-server generate-ts --experimental` 在本机 `0.144.4` 上成功，生成 671 个类型文件。
- stdio 的 `initialize` / `initialized` 握手已对 `0.144.4` 实际通过。
- 当前协议有 11 类服务端主动请求，不只包括执行、改文件、权限和 elicitation；还包括动态工具调用、认证令牌刷新、attestation、当前时间及两类旧版审批。
- 本机同时存在 `0.144.4` PATH CLI 与 `0.144.2` 桌面内置 CLI，因此不能按“命令可启动”判断兼容。

## 实施边界

- Rust 只在用户选择的本地目录托管每会话一个 `codex app-server --stdio`、JSONL 输入输出、退出和强制终止；不解释 Agent 业务协议，也不开放任意命令执行入口。
- TypeScript 负责生成协议类型、请求关联、会话状态机、审批策略、聊天上下文打包和 UI。
- 每个话题会话使用独立本机 stdio 进程和所选工作目录；不引入 daemon/proxy。
- 不预先抽象多 Agent Provider；M8 直接绑定 Codex app-server。
- 会话元数据复用现有 `appData`，使用稳定前缀隔离，不新增 IndexedDB schema。
- Bot 账号优先；没有 Bot 配置时允许宿主账号降级发送。消息展示标记不参与审批和信任判断。

## 执行顺序

1. **协议锁定**：提交完整生成类型、版本清单和再生成/漂移检查脚本。验证：初始化握手测试、11 类请求穷举测试、未知请求拒绝测试。
2. **进程宿主**：新增 `proc.rs`，提供发现、启动、写入、终止和事件桥。验证：Rust 单测、真实启动、kill 后退出事件。
3. **协议客户端与状态机**：实现 JSON-RPC 关联、turn 串行、thread start/resume、租约和中断恢复。验证：纯单测覆盖正常、拒绝、超时、崩溃和 resume。
4. **审批与安全**：把 app-server 请求映射为主持人审批，加入白名单、敏感路径拒绝、外部副作用限制和输出脱敏。验证：`.env` 拒绝、密钥样式脱敏、非主持人不能批准执行。
5. **话题集成与 Agent 面板**：订阅话题、识别触发，打包话题/引用线程/参与者/ADO/只读附件上下文，完整回复回帖，展示队列/过程/审批/中断状态。验证：宿主与第二成员各完成一轮查阅，执行请求等待宿主。
6. **反向 MCP**：实现受范围约束的 `rcx-mcp` stdio server，提供会话列表、话题上下文和房间历史读取。验证：外部 Codex 连接后能读取允许范围，越权范围被拒绝。
7. **端到端验收**：真实 Rocket.Chat + Tauri + Codex 走完整修复流程，并用官方客户端旁观；随后运行 typecheck、纯测试、回归、smoke、Rust 测试和生产构建。

## 已关闭门禁

- 用户明确选择本机目录模式；独立 Codex 与共享 Agent 均使用 Codex 原生 OS sandbox，不再构建或运行 Agent Runner 容器。
- Codex CLI 缺失、版本不兼容或未登录时安全失败；Docker 只服务于可选的 Rocket.Chat 自部署栈。

## 风险与回退条件

- app-server 是 experimental：初始化返回版本与锁定矩阵不匹配时拒绝进入可执行态，只允许显示诊断和重新生成指引。
- Rocket.Chat 自定义字段不保证保留：共享会话事实以普通话题消息和宿主本地状态为准，字段只做渐进增强。
- 多设备租约依赖可见心跳消息：若服务端更新语义无法可靠保持单写者，M8 不宣称自动接管，改为明确超时后人工接管。
- 反向 MCP 不能直接读取 WebView 内存：使用最小本地桥接和账号作用域存储，不复制 Rocket.Chat 凭据到命令行参数或日志。
