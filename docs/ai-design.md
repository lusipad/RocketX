## 一、管家是什么

好管家的本质不是能力，是**判断力**。

能力是"能帮你做什么"——总结、提醒、排日程。这是工具属性。
判断力是"知道此刻该不该做、做到什么程度、以什么姿态出现"。这是管家和工具的分水岭。

v3 补充一条更硬的定义：**管家的产品不是提醒，是替你做出的注意力决定——其中九成的决定
是"这事不值得你知道"**。因此 KPI 不是覆盖率（盯到了多少事），是**信噪比**（每一条都值得
你抬头）。每条产出必须通过闭嘴测试：

> **"如果我闭嘴，你会漏，且漏了你会后悔" → 才说；否则静默。**

四条性质定义一个合格的管家：

| 性质 | 含义 |
|------|------|
| **懂分寸** | 忙时不打扰，该提醒时在恰当时机说，不刷存在感 |
| **主动但不越界** | 发现冲突会提前说并给方案，但决定权永远在你手里；不可逆操作必须确认 |
| **记性好但嘴严** | 记得你的习惯、承诺、偏好，但这些信息只为你服务，随时可查看和删除 |
| **诚实** | 不知道就说不知道，办砸了就承认；沉默必须可审计 |

## 二、v2 为什么失败（诊断，防止重蹈）

v2 的判断层 = 确定性规则引擎（`butlerRules.ts`），判据写的是"漏了有后果才说"，但规则
只能实现"字段匹配就说"：

- **事件思维**：构建规则逐条 build 报警，从不检查同一流水线最新一次是否已通过——被修复的
  失败永远尖叫；工作项规则只看 `changedDate` 跳没跳——改标签、改区域都触发。
- **判断力被禁**：v2 §九明确"不做 AI 判断消息重要性"（理由：不可预测、不可解释）。结果是
  "有没有后果"这个本质上的判断题，被硬编码成 if-else，没人有资格回收错报。
- **代价**（警报工程研究的标准结论）：误报的真实代价不是这一条，是**贬损所有未来警报**
  ——狼来了效应把用户训练成"这页反正没用"。信任毁掉后极难重建。

结论：原则（闭嘴测试）是对的，实现手段（纯规则）与原则矛盾。v3 换模型，不再修补规则。

## 三、核心模型：对账员

一句话定位：**管家 = 注意力守门人 + 承诺记账员。它不是 ADO/IM 的镜子，它的本事是决定
不报什么。**

核心循环从"过滤事件流"改成"**对账**"：

```
感知世界当前状态 → 与台账对账 → 只有差异才值得思考 → 思考产出 = 简报 / 台账变更提议 / 极少数打断
```

三条铁律：

1. **判断当前状态，不判断历史事件。** 事件（失败过、改过）不进警报；巡视只看"现在什么
   是真的"。自愈是免费属性：不再成立的事实下一轮对账自动消失——"CI 跪了又绿"这类噪音
   按构造不存在，无需任何抑制规则。
2. **台账是判断的地基。** 一件事值不值得说，取决于它碰没碰台账（你答应的、你在等的、
   你今天想推进的）。"重要性"不再凭空打分。
3. **传感器可以叫醒大脑，但只有大脑能开口。** 确定性代码负责感知与叫醒；说不说、怎么说
   永远是大脑（巡视）的判断。唯一例外见 §六 安全网。

对账矩阵（世界状态 × 台账 → 动作）：

| 世界状态 | 台账 | 对账结果 |
|---|---|---|
| 流水线最新一次 = 绿 | 无关 | 无差异 → 沉默（历史失败根本不出现） |
| 流水线最新一次 = 红 | "今天要合 PR #123" | 台账受阻 → 上简报，给动作 |
| 工作项改了标签 | 不碰任何条目 | 无差异 → 沉默（只进日志计数） |
| 新工作项指派给我 | 台账不认识它 | 差异 → **提议入账**，不是警报 |
| 承诺今天到期未动 | 台账核心条目 | 差异且不可逆 → 打断（安全网，见 §六） |
| 我等的人回复了 | 有对应等待条目 | 差异 → 提议销账 + 简报告知 |

## 四、台账

**承诺不是独立子系统，是待办的一种**（继承 v2，数据结构已在 todos store 的
`committedTo` / `waitingFor` / `due` 字段）。台账 = 从 todos 派生的视图 + 提议队列，
**不建平行存储，不双写**。

- 条目两类：**我答应的**（committedTo）、**我在等的**（waitingFor）。
- **极小化**：只收承诺与等待，绝不收全部任务——台账维护疲劳是 GTD 工具的头号死因，
  条目越少账越准。
- **管家负责提议开与关**：看到新指派 → 提议入账；看到 PR 合了 → 提议销账。全部提议制，
  用户一键确认/拒绝，管家永不自动写账。
- **容忍陈旧**：拿不准台账是否过期时问，绝不基于可能过期的台账替用户拿主意。
- 「今日意图」实体暂缓：先让大脑从 due-today + 活跃工作项推断当日焦点，dogfooding
  证明推断不准再立实体。

心理学依据（承诺追踪为什么是杀手能力）：未完成之事持续占用后台注意力（Zeigarnik 效应）；
且**只要被可靠记下并有计划，焦虑即解除**（Masicampo & Baumeister 2011）——所以台账的
成败标准是"**可信地一件不漏**"，不是"提醒得多巧"。漏一次，用户就不敢把事放下，整个
功能作废。

## 五、巡视（判断层的新实现）

管家的大脑不做流式过滤，做**定期查房**：拿当前世界快照 + 台账，按闭嘴测试逐项对账，
产出简报。

**节奏**（大脑一天跑 3~5 轮，不随 10 分钟轮询烧）：

| 触发 | 阶段 |
|---|---|
| 每日首次打开应用 | 一期 |
| 桌面手动"看一圈"按钮 | 一期 |
| 离开 > 2h 后回来 | 二期 |
| 晚间收尾（默认 18:00，可做成例行事务） | 二期 |
| 传感器叫醒条件命中（如"我的活跃 PR 最新构建变红"）→ 触发提前巡视 | 二期 |

**大脑路由**：遵守决策 11 双大脑。巡视走一次性上下文（Codex = ephemeral 线程 +
medium 档；API = AiBus）。把 `runButlerCodexEphemeral` 包成 `AiChatGateway` 适配器
（文本 → 单 chunk + finishReason），**一份巡视实现同时服务两个大脑**；显式开关，
不静默回退。

**输出契约**（代码强制，复用 daily-review 的结构化输出 + 引用校验模式）：

```
{ brief:      { headline, summary, items: [{ ref, why, suggestedAction? }] },   // 硬上限 3 条
  proposals:  [{ kind: 'add-commitment' | 'close-wait' | …, ref, reason }],     // 世界 vs 台账差异
  suppressed: [{ ref, reason }] }                                               // 看过没提的 → 工作日志
```

- 每个 `ref` 必须存在于输入快照，否则整轮作废（反幻觉）。
- **配不出建议动作的条目自动降入日志**——"说不出该做什么的事没资格打断人"（SRE 告警
  铁律：page/ticket/log 分层）。
- **契约在代码，判断在文本**：什么算值得说、语气分寸，写在管家默认技能文本里
  （SKILL.md，对话即配置）；用户调教改文字，不改代码。
- 晨报/晚回顾（v2 咖啡时间）被巡视吸收：9:00/18:00 只是两个预定巡视点。

## 六、传感器（v2 感知层降级更名）

保留 v2 的全部感知基建，剥夺其开口权：

- **客户端轮询不变**：Tauri 常驻、默认 10 分钟、ADO 只读标准字段（Assigned To / State /
  Priority / Iteration / Changed Date）、不写回、不扫全组织、无后台服务、关了就停。
- 轮询只做三件事：刷新快照、维护台账提议的原料、检查安全网与叫醒条件。
- **安全网（唯一保留的规则警报）**：承诺日期到期/逾期 → 实时系统通知。这是纯日期算术，
  确定、可测、不需要判断，也不该等巡视。复用现有通知链路（rid='butler'）。
- **删除的规则**：新高优工作项、构建失败、review 超时、迭代压力——原始事实全部改为
  进巡视快照，由大脑对账后决定说不说。
- 快照序列化时构建按 `definition + project` 归并只取最新一次——这是传感器卫生
  （事实清洁），不是判断。

## 七、桌面（表达层的新家）

> 交互原型：`docs/prototypes/butler-desk.html`（四个时刻可切换，日志/入账/销账/停靠对话可点；
> 线上版 https://claude.ai/code/artifact/9e742a54-1afa-4fd6-87f0-cb641921e000 ）。
> 原型演示完整愿景，含二三期交互；一期落地范围以 §13 为准。

管家页 = 管家的桌面，四段，自上而下：

| 段 | 内容 | 规矩 |
|---|---|---|
| **简报** | 最近一轮巡视产出，≤3 条，每条带"为什么找你"+ 一键动作 | 头条可红（承诺类），其余平静 |
| **台账** | 我答应的 / 我在等的 两列，含到期状态 | 可确认提议、可销账 |
| **工作日志** | 默认折叠。"看了 12 个工作项、5 个 PR、3 条流水线，上简报 3 条"+ 被压条目及理由 | 沉默的可审计凭据，信任靠它挣 |
| **停靠对话** | 底部输入框，一拉展开为完整管家对话 | 同一常驻线程 |

- **简报和台账是状态，聊天是流**——所以桌面为体、对话停靠，绝不把简报做成聊天气泡
  （会被流冲走，台账被冲走等于管家失忆）。
- **原始清单退场**：未完成工作项 / 待 review PR 的平铺列表归工作台模块，管家页不再当
  镜子。
- **三面合一**：`today`（今日）、`butler-view`（管家页）、`ai-assistant`（管家对话）
  三个入口合并为唯一"管家"入口；例行事务 `delivery:'today'` 的投递位改渲染在桌面。
- 错过巡视（客户端关着）：下次打开自动补一轮。

## 八、唯一实体与入口

M12 第一原则的落实检验：**用户处处见到的必须是可验证的同一个人**。

器官版图（用户视角）：

| 表面 | 用户何时知道 | 理解成本 |
|---|---|---|
| 桌面（导航"管家"一格） | 第一天 | 零——就是一个联系人 |
| 房间内召唤（顶部按钮 / `/ai`） | 用到时自然发现 | 零——"叫管家过来看这个群"，IM 肌肉记忆 |
| 执行间（Ctrl+K） | 永远不必知道 | 藏于导航之外；它是机器不是人的房间 |
| 消息上的随手动作（提取待办等） | 用到时 | 不冠"AI"之名——它们是管家的反射，产出走台账提议 |

**四条"同一个人"不变量**（任何新表面上线前的验收标准）：

1. **同名同头像**——桌面、面板、系统通知署名全是"管家"。
2. **一场对话**——房间里说半句，回桌面接着说，历史连续（同一常驻线程 +
   `roomPrefixedInput` 已保证）。
3. **一份记忆**——面板里说"少管迭代的事"，明早简报就得记得。
4. **一个通知来源**——永不出现"系统提醒"与"管家提醒"两个声音。

**UI 语言纪律**：界面只说人话——"我答应的 / 我在等的 / 这 14 条我看过，没提"。
架构词（对账、台账、传感器、巡视轮次、大脑、ephemeral）一个不许漏进 UI；"已巡视"
写成"我看了一圈"。管家自称"我"，不自称"系统"。

## 九、性格与表达（继承 v2）

四轴模型不变，人格在表达层生效，影响简报措辞与打断姿态，**无权改变对账结论与契约**：

| 轴 | 低端 | 高端 |
|----|------|------|
| 话量 | 一个标记 | 主动说一句话 |
| 介入深度 | 只标记问题 | 标记 + 方案选项 |
| 语气 | 正式克制 | 轻松随意 |
| 催促感 | 纯信息陈列 | 带轻微紧迫感 |

默认人格：克制、预判、不越界（英式管家）。实现 = persona prompt + 轴参数
（复用现有 `butlerPersonality` / `butlerExpression` / `buildButlerSystemPrompt`）。

## 十、信任冷启动与打断权

用户已被 v2 训练到无视管家页，恢复期必须**过度安静**（警报疲劳不可逆，重建从超保守起步）：

- 头两周：实时打断**只有承诺到期一种**；其余一切进简报。
- 工作日志第一天就可见——管家先用日志证明"我压掉的都对"，再谈更多打断权。
  放权是挣来的，不是默认的。
- 每条产出必带"为什么找你"；用户点一次"少来这种"→ 走记忆管线（`remember`），
  下轮巡视召回。
- 注意力预算原则：实时打断有日上限，超出自动降入简报。

## 十一、三个赌注与证伪标准（诚实清单）

本设计是三个假设的实验，不是定稿蓝图。每个赌注都有便宜的证伪信号和降级出口：

| 赌注 | 证伪信号 | 降级出口 |
|---|---|---|
| 台账不会变成维护负担 | 一周后提议确认开始积压、用户懒得点 | 台账缩到只剩承诺（committedTo），等待类全自动推断不上账 |
| 大脑判断力真的好（不戏精、不贪多、轮间稳定） | 简报出现"戏剧化小事"；同一事项上榜下榜反复 | 简报退化为安静晨报（只做汇总不做研判），架构不塌 |
| v1 薄输入也有价值 | 只有 todos+ADO 的简报显得空洞 | 提前拉入日历冲突与 @我 未回信号 |

注意预期：桌面 mockup 里"上周五你在群里答应的"这类句子是**三期**（聊天扫描）的能力；
一期的"为什么"只能引用你亲手录的台账与 ADO 事实。

最坏结局 = 一个不吵的晨报 + 一本承诺账——仍然优于 v2；最好结局 = 会替你操心的人。
下行有底，上行够高。

## 十二、明确不做（v3 修订版）

**继续不做**：

| 不做 | 为什么 |
|------|--------|
| 后台服务 / bot 化 | 客户端轮询够用；你不在电脑前时不需要被提醒 |
| 全组织扫描 | 只看配置的项目（隐私 + 性能） |
| 写回 ADO | 管家是安静的读者 |
| 多套人格卡片 | 轴参数已覆盖 |
| AI 排全量任务优先级 | 待办排序仍走规则（到期日 + 手动优先级），可预测 |
| 简报做成推送流 | 状态不进流；桌面已是唯一载体 |
| 事件警报管道复活 | 任何"新增一条规则警报"的提案都必须先过 §三铁律 |

**v2 拒绝、v3 有条件推翻**（标注理由，防止未来误读为摇摆）：

| v2 的"不做" | v2 理由 | v3 处理 |
|---|---|---|
| AI 判断该不该提醒 | 不可预测、不可解释 | **推翻**。dogfooding 证明没有判断力才不可用。可解释性改由结构解决：契约 + 引用校验 + 每条带 why + 日志可审计 |
| AI 从聊天识别承诺 | 猜错清理成本高 | **三期以提议制引入**：只提议不入账，错了成本 = 点一次忽略 |

**冻结区**：聊天扫描（一二期不做）、今日意图实体（视 dogfooding）、技能市场、多人设、
向量化情景记忆（沿用 blueprint §11）。

## 十三、落地顺序（三阶段，每段独立可验收）

### 阶段一 · 换心脏（巡视上线，警报墙拆除）

| 动作 | 落点 |
|---|---|
| 台账派生视图 | 新 `apps/web/src/lib/butlerLedger.ts`（从 todos 派生，零新存储） |
| 巡视 feature | 新 `apps/web/src/kernel/ai/features/butler-rounds.ts`（照 daily-review 模式）；`AiCapability` 增 `'butler-rounds'` + 路由注册 |
| Codex 适配器 | `runButlerCodexEphemeral` → `AiChatGateway` 包装 |
| 传感器降级 | `butlerRules.ts` 砍到只剩承诺日期安全网；`butlerPoller.ts` 去掉警报派发（承诺除外） |
| 桌面重建 | `ButlerPage.tsx` 改四段布局；链接墙退场 |
| 测试 | 新 `butler-rounds.test.ts`（假 gateway 契约测试）；重写 `butler-poller-rules.test.ts` |

**验收**：CI 绿掉的历史失败、纯标签变更在任何地方不再出现；简报 ≤3 条且每条有 why；
安全网通知仍达。

### 阶段二 · 长记性（台账闭环 + 反馈）

提议真入账（契约扩展 who/due，`add-commitment` 写 committedTo、`schedule-today` 落
今日、`close-wait` 销账）→ "少来这种"（提示词注入偏好 + **代码侧强制降级**：命中静音的
条目由代码降入日志并标注理由，不赌大脑听话；静音列表可查看可删）→ 离开 >2h 回来 /
晚间收尾 / 传感器叫醒条件三类触发（叫醒只触发提前巡视，绝不直接通知）。
**验收**：新指派走提议且入账真落库；说过"少来 X"后 X 被强制降入日志；三类触发各自
当天不重复。

### 阶段三 · 合桌子（唯一实体收官 + 放权起步）

导航收编（`today` / `butler-view` / `ai-assistant` → 管家，`runtime.tsx` 模块注册 +
navigation/semantic-entry 回归迁移）→ 今日并入桌面（`delivery:'today'` 渲染位迁移，

（转任务 / 起草回复 / 稍后）。
**验收**：一级导航一个管家；"我明天发你"出现后管家提议记承诺；四条不变量全过。

## 十四、一句话

管家看得见你的工作全貌，但它真正的本事是**替你决定什么不值得看**——它拿每一轮世界的
真实状态对你的承诺账本，差异才开口，开口必带理由，沉默全可审计；一天里它只在你要对
别人失信时打断你一次，其余时间，它安静地把"没什么需要你"这份底气端给你。

## 附 · 研究依据（一行一条）

- 警报疲劳 / 狼来了：临床警报 85–99% 无需处理 → 真警报也被无视（Joint Commission 2013；Breznitz 1984）——误报贬损全部未来警报。
- 中断期望价值：打断当且仅当信息价值 > 打断代价（Horvitz, Mixed-Initiative UI, 1999）；Clippy 是违背它的反面教材。
- Zeigarnik 效应 + Masicampo & Baumeister 2011：未完成事项持续占用心智；**可靠记录 + 有计划即解除**——台账的完整性 > 聪明。
- 注意力恢复成本：被打断后平均 ~23 分钟回到原任务（Gloria Mark）；注意力残留（Leroy）——打断汇聚到自然断点（bounded deferral, Horvitz）。
- SRE 告警哲学：说不出"该做什么"的没资格打断（page/ticket/log 分层，Rob Ewaschuk）。
- 实施意图（Gollwitzer）：给"下一个具体动作"而非"任务"。
- 自动化信任校准（Lee & See 2004；Parasuraman）：误报导致 disuse 后极难挽回 → 冷启动过度保守；同时防自动化自满——保留用户对全局的掌控感。
