# 更新日志

## v1.0.0 - 待发布

### 新增

- 完成 M11 应用生态：`@rcx/app-sdk` 成为 manifest 与 Bridge 的公开类型契约，`create-rcx-app` / `rcx-app` 提供脚手架、校验和回环地址开发预览。
- 官方样板扩展为看板、投票和值班表；全部只使用公开能力和最小权限，不依赖宿主私有 API。
- 新增固定版本的一键 Web 自部署栈，包含 RocketX、Rocket.Chat 8.6.1 与 MongoDB 7.0.21，并提供同源 API、WebSocket、文件和头像反代。
- 补齐英文主 README、中文入口、应用开发、兼容矩阵、贡献、安全与第三方许可文档。

### 发布工程

- 统一仓库、Web、桌面、SDK、CLI 与桥接服务版本为 1.0.0，并把 Codex 0.144.4 的 671 个协议生成文件纳入 CI 一致性门禁。
- 三平台 Release 工作流先执行协议、类型、测试、应用生态和 Web 构建门禁，再核验 Windows、macOS、Linux 产物并生成 `SHA256SUMS.txt` 后发布。
- 本地 clean-room 应用生态、三服务 Docker、真实 Rocket.Chat、Codex app-server、Agent Runner、Rust 和 Windows Release 已通过；正式标签仍以外部开发者 G3/G4 真人计时及三平台 CI 为门禁。

### 安全

- CLI 开发服务只绑定回环地址；应用 manifest、entry 路径和权限继续由 SDK 单一解析器校验，不为开发模式放宽宿主 CSP。
- Compose 明确定位为本地/评估基线；生产部署仍必须配置 TLS、强凭据、网络收敛与备份恢复。

## v0.19.0 - 2026-07-17

### 新增

- 完成 M10 IP Messenger 共存模式：桌面端按需监听 UDP/TCP 2425，支持成员发现、普通消息确认重试和普通文件拉取。
- 消息侧栏新增独立的本地兼容频道，可选择在线 IP Messenger 联系人发送消息或文件；局域网消息同时进入「今日」收件箱。
- 设置页新增默认关闭的兼容模式开关、端口冲突错误和在线联系人状态，不影响 M9 可信 LAN。
- 原生协议支持标准 UTF-8/CP932 与飞秋 GBK fixture；官方 IP Messenger 5.8.3 已完成真实发现、消息和文件互通验收。

### 安全

- IPMSG peer 明确标记为未认证旧协议身份，不复用 M9 固定公钥、可信设备或 Rocket.Chat 会话动作。
- 收到的文件必须由用户显式下载，只写入应用数据目录；文件名拒绝路径穿越，临时文件完整接收后才发布。
- 飞秋公开二进制因无签名且当前宿主缺少可用隔离环境而未执行；按蓝图收缩为原版 IP Messenger 真实互通，不虚报飞秋运行时兼容。

## v0.18.0 - 2026-07-17

### 新增

- 完成 M9 可信局域网：mDNS 主发现、TTL=1 的 UDP 组播兜底、Rocket.Chat 认证通道设备公钥交换，以及 Ed25519 固定公钥双向挑战应答。
- 桌面端文件发送在可信单人会话中自动使用原生 TCP；固定 1 MiB 分片、最多四路并发、缺块续传、分片和整文件 BLAKE3 校验，并显示实测速率。
- Rocket.Chat 不可达时，普通消息可经可信 LAN 投递；作者使用稳定消息 ID 从 IndexedDB outbox 幂等回灌，参与会话的设备保留原始时间。
- 应用能力总线新增 `lan.peers` 与 `lan.send`，分别受 `lan:discover`、`lan:transfer` 权限控制；peer 输出不包含 IP、端口或设备公钥。
- 桌面设置新增 LAN 与离线回灌诊断，明确当前可信设备数量及服务器消息自定义字段能力。

### 安全

- 广播中的 userId、公钥和地址永远只作为发现候选；未在 Rocket.Chat 认证会话中固定公钥的设备不能建立 LAN 连接。
- LAN 文件名拒绝路径穿越，接收文件只写应用数据目录；每片校验后写入临时文件，整文件校验通过才发布。
- Rocket.Chat 回退上传前读取服务器文件大小上限，避免把超限大文件整体载入 WebView 内存。

## v0.17.0 - 2026-07-17

### 新增

- 完成 M8 共享 Agent：Rocket.Chat 线程直接承载 Codex 长会话，支持会话租约、恢复、串行命令队列、成员加入审批和本地过程面板。
- Agent 上下文新增引用消息整条线程、参与者、关联 ADO 工作项和只读临时附件；宿主可切换“仅自己”并用 `/exit` 结束会话。
- 接入锁定版本的 Codex app-server 协议，生成类型、真实 stdio 握手、流式回复、服务端主动请求与进程中断均由薄适配层处理。
- 新增只读优先的工作区安全边界、显式敏感路径请求拒绝、输出密钥脱敏和危险动作宿主审批。
- 新增每会话独立的 Docker Agent Runner；仅挂载当前工作区、只读上下文附件和只读 Codex 认证文件，支持状态诊断、一键安装与容器终止后的 thread resume。
- 新增可选 Agent Bot 身份；凭据仅存系统凭据库，且只能向配置所属的 Rocket.Chat 服务器发送。
- 新增反向 MCP stdio 入口，向外部 Agent 提供会话列表、线程上下文和房间历史三个只读工具。

### 安全

- Codex 默认运行在固定 Linux Runner 的只读权限配置中；`.env`、密钥目录和认证文件由 bubblewrap 拒绝，写工作区、执行命令及其他高影响请求仍须当前宿主批准。
- 反向 MCP 与 Agent Bot 的 Rocket.Chat token 不接受命令行、环境变量或明文配置文件输入，状态接口也不返回 token。

## v0.12.1 - 2026-07-15

### 改进

- 快捷搜索不再因空结果或实时会话更新向 Rocket.Chat 连续发送逐房间请求，并限制回退搜索的并发量。
- 通讯录与房间成员分页增加人数、页数和无进展保护；多个面板同时读取成员时复用同一请求。
- 46 项并发、分页、信任边界与状态隔离回归测试纳入持续集成。

### 修复

- 修复快捷搜索无法按拼音找到中文联系人，以及较早搜索结果覆盖当前关键词的问题。
- 修复通讯录重复页、成员列表重复页或异常总数可能造成长时间请求和残缺结果的问题。
- 修复切换房间后旧成员请求回写当前面板，以及邀请成员时旧快照覆盖新名单的问题。
- 修复切换 Azure DevOps 服务器时旧刷新覆盖新配置、数据和错误状态的问题。
- 修复切换或清空 Azure DevOps 连接后，工作项链接仍可能指向旧服务器的问题。
- 修复多个 Azure DevOps 自定义查询共用加载、错误和缓存状态的问题。
- 修复自定义 Azure DevOps 查询跨账号或服务器复用、外部伪装链接可被保存，以及升级迁移可能误删其他服务器旧查询的问题。
- 修复邀请群外成员期间切换会话时，消息可能被发送到新会话的问题。
- 修复搜索关键词变化时短暂显示上一关键词结果，以及部分重复分页静默缺少用户的问题。
- 修复快速变化未读数时，较慢的旧任务栏角标最后覆盖最新值的问题。

## v0.12.0 - 2026-07-15

### 新增

- 快捷搜索同时覆盖会话、跨会话消息、联系人和频道，并支持中文拼音与备注名。
- 通讯录可完整加载 Rocket.Chat 用户目录，成员与文件列表支持服务端分页。
- 从聊天创建 Azure DevOps 工作项时，按项目过程动态加载真实工作项类型。

### 改进

- 优化消息列表滚动锚点、窄窗口布局、线程与侧栏高度、PDF 预览资源回收。
- 完善合并转发和跨房间附件处理，保留可复用内容并阻止原房间受保护文件链接泄露。
- Azure DevOps Bridge 自动识别当前身份，并按当前用户筛选工作项、PR 与构建。
- 统一模态弹窗的 Esc、焦点陷阱和焦点恢复行为。

### 修复

- 修复 Rocket.Chat 文件重定向到外部地址时可能继续携带认证头的问题。
- 修复成员邀请后被较早请求的旧名单覆盖，以及刷新失败时无法可靠重试的问题。
- 修复 Azure DevOps Server 2022 不存在的截止日期字段导致 Bridge 请求失败的问题。
- 修复工作项类型加载失败后仍可使用陈旧类型创建、错误配置被误报为连接成功的问题。
- 修复实时消息更新反复重启快捷搜索，以及创建工作项过程中关闭弹窗可能造成重复提交的问题。
- 修复收藏链接协议校验、弹窗键盘操作、窄视口和多项工作台数据边界。
