# 更新日志

## v0.14.11 - 2026-07-17

### 新增

- 消息搜索提供明确的“搜索全部”入口；需要完整历史时可主动扩大到所有会话，并继续向下滚动加载更早结果。

### 优化

- 输入关键词时默认先搜索本机已加载消息和当前会话，避免每次搜索都立即扫描全部会话，降低等待时间和 Rocket.Chat 服务端压力。
- 搜索范围、加载进度和完成状态会直接显示在面板中；修改关键词后自动回到轻量搜索，不会沿用上一次全量范围。

### 文档

- 产品蓝图更新为 v2，以 GTD 与注意力保护为双支柱，重新定义 M6~M11、稳定化轨道和开源发布门禁。

## v0.14.10 - 2026-07-17

### 优化

- 消息搜索首批显示 20 条，滚动到底后自动加载下一批；服务端全局搜索按 20 条递增请求，逐会话回退使用 `offset` 分页，不再把 20 条作为总结果上限。
- 搜索分页结果缓存从最近 20 个查询收敛到 5 个，超过 200 条的首批结果不跨查询缓存，避免单次查询结果增多后放大内存占用。

### 修复

- 修复 Windows 桌面端导出诊断日志时被 Tauri 文件命令 ACL 拒绝的问题；继续沿用现有文件写入权限，不扩大文件系统授权范围。

## v0.14.9 - 2026-07-16

### 优化

- 全局搜索会先从本机已加载消息中立即返回结果，再按最近会话优先分批补全完整历史，不再等待所有会话搜索结束后才显示消息。
- 搜索状态分别提示联系人/频道和完整历史消息，远端历史搜索失败时仍保留本机已找到的结果。

### 修复

- 已退出或已删除的私有会话不会通过本机消息缓存重新出现在搜索结果中。

## v0.14.8 - 2026-07-16

### 新增

- Azure DevOps 工作项列表支持按当前查询结果中的父子关系分层显示，并可逐级展开或折叠；搜索和状态筛选会保留命中项的祖先路径。

### 修复

- 修复从搜索结果或通知定位过旧消息后，重新打开会话可能先滚到最新消息、随后又向上回弹的问题。
- 修复 Azure DevOps 树查询中重复关系可能占用结果上限、漏掉后续唯一工作项的问题。

## v0.14.7 - 2026-07-16

### 修复

- 修复部分 Rocket.Chat 服务器未开启全局消息搜索时，快捷搜索会把服务端空结果误判为“没有相关消息”的问题。
- 全局搜索不可用时自动低并发查找全部可访问会话的完整历史，只在客户端保留最近 20 条候选；相同范围的成功查询会短时复用，兼顾搜索完整性与资源占用。

## v0.14.6 - 2026-07-16

### 修复

- 修复打开或重复点击会话时，消息列表可能在布局稳定前完成首次滚动，导致没有停在最新消息的问题。
- 消息内容异步撑高、输入区或窗口改变滚动视口高度时，继续保持最新消息可见。

## v0.14.5 - 2026-07-16

### 改进

- 长时间运行时只完整保留当前和最近使用的会话数据，较早会话保留消息预览并在再次打开时重新加载；Azure DevOps 悬停查询也增加容量和时效边界，减少持续增长的内存占用。
- 通讯录、成员列表和搜索结果中的离屏头像改为接近视口后再加载，头像与聊天图片使用独立缓存，减少无效请求和相互淘汰。

### 修复

- 修复从消息切到大通讯录再返回后，常用头像可能变成空白或需要滚动才重新出现的问题。
- 修复切换 Rocket.Chat 服务器或账号时，旧会话迟到的历史、角色或回执响应可能写入新会话状态的问题。

## v0.14.4 - 2026-07-16

### 新增

- 隐藏会话增加独立分组，可直接恢复并打开，不再只能等待新消息重新出现。

### 改进

- 默认发送方式改为 Enter 发送、Alt+Enter 换行；设置中的 Ctrl+Enter 发送模式继续保留。
- 多选态增加明确的消息勾选按钮，批量删除只处理自己的消息并按顺序完成。
- 消息操作统一显示“删除”，不再按消息时间在“撤回”和“删除”之间切换。

### 修复

- 修复多选消息点击不稳定，以及并发批量删除可能让已删除消息重新出现的问题。
- 修复引用图片在 Rocket.Chat 服务端展开为嵌套附件后不显示的问题。

## v0.14.3 - 2026-07-16

### 新增

- 消息多选工具栏支持直接复制、逐条转发、合并转发和 Markdown 导出；文字、图片与文件说明共用同一输出规则。
- 通知设置增加独立的任务栏窗口闪烁开关，不再与托盘图标闪烁混为一项。

### 改进

- 全局搜索的“全部”结果把联系人和频道置于最上方，减少从搜索发起会话的操作距离。
- 桌面头像 objectURL 缓存限制为 128 项并安全回收未使用资源；切换会话时只保留当前房间的实时订阅，降低长期运行的 WebView2 内存与事件分发压力。

### 修复

- 修复多选状态提示可按 Esc 取消、但实际没有响应 Esc 的问题。
- 修复附件消息无法通过单条“复制”操作输出内容的问题。

## v0.14.0 - 2026-07-16

### 新增

- 新用户可通过可恢复的首次引导完成 Rocket.Chat 登录和 Azure DevOps 配置；ADO 暂不可用时可跳过，进入主界面后继续完成会话、消息与通知清单。
- Windows 全局快捷键可从任意位置打开统一指令中心，优先处理未读，并搜索会话、消息、联系人、文件和工作数据。
- 搜索支持按发送人、时间和文件类型筛选；常用文件使用有边界的本地索引，不需要逐个会话发起请求。
- Windows 桌面端支持开机自启、单实例唤回、通知点击直达会话，以及脱敏的本地诊断日志导出。
- 会话列表宽度、分组栏和高频快捷键可按个人习惯调整，并支持连续跳转到下一条未读。

### 改进

- 通知权限改为用户主动启用，不再在登录后静默申请；首用清单可永久跳过且不会重复提醒。
- 创建 Azure DevOps 工作项时会记住当前账号和连接最近选择的项目，复用现有层级模板。
- Azure DevOps PR 卡片展示发起人、评审进度和服务端标记的必审组；“我提的 PR”只显示活动项。
- 全局搜索优先使用 Rocket.Chat 服务端能力，并在不可用时以受控并发回退，避免请求风暴和旧结果覆盖新查询。

### 修复

- 修复切换 Rocket.Chat 服务器或账号后，引导、布局和 Azure DevOps 项目选择可能串用的问题。
- 修复桌面通知导航接受无效房间标识、重复启动产生多个窗口，以及较早搜索请求覆盖当前结果的问题。

## v0.13.0 - 2026-07-16

### 新增

- Azure DevOps PR 与构建链接可在消息中直接展开为摘要卡片；加载失败时仍保留可点击的原链接。
- Windows 托盘图标会在有未读消息时闪烁，并在未读清零、退出或重载时可靠恢复。

### 改进

- 层级工作项按 Azure DevOps Server 的真实过程配置生成，兼容 Basic、Scrum、CMMI 与自定义流程。
- 未加入的公开讨论仍可查看和发送消息；加入后才启用订阅通知与已读同步。
- 首次打开会话默认滚动到最新消息，未读分割线仅作视觉提示。
- 任务栏未读提醒不再依赖桌面通知开关，并继续遵守免打扰与当前窗口焦点状态。

### 修复

- 修复公开讨论误用频道加入接口导致 `error-room-not-found` 的问题；旧版服务端仅在 `rooms.join` 返回 404 时回退 DDP。
- 修复已有讨论订阅未及时刷新、进入讨论后未读无法清除，以及未订阅讨论错误触发通知的问题。
- 修复分页首条被误判为未读边界、托盘异步旧帧覆盖正常图标等竞态。
- 78 项专项回归纳入持续集成，覆盖 ADO 卡片、讨论访问、初始滚动、任务栏与托盘状态。

- 快捷搜索不再因空结果或实时会话更新向 Rocket.Chat 连续发送逐房间请求，并限制回退搜索的并发量。
- 通讯录与房间成员分页增加人数、页数和无进展保护；多个面板同时读取成员时复用同一请求。
- 46 项并发、分页、信任边界与状态隔离回归测试纳入持续集成。

### 修复

- 修复快捷搜索无法按拼音找到中文联系人，以及较早搜索结果覆盖当前关键词的问题。
- 修复通讯录重复页、成员列表重复页或异常总数可能造成长时间请求和残缺结果的问题。
- 修复切换房间后旧成员请求回写当前面板，以及邀请成员时旧快照覆盖新名单的问题。
- 修复切换 Azure DevOps 服务器时旧刷新覆盖新配置、数据和错误状态的问题。
- 修复切换或清空 Azure DevOps 连接后，工作项链接仍可能指向旧服务器的问题。
- 修复多个 Azure DevOps 自定义查询共用加载、错误和缓存状态的问题。
- 修复自定义 Azure DevOps 查询跨账号或服务器复用、外部伪装链接可被保存，以及升级迁移可能误删其他服务器旧查询的问题。
- 修复邀请群外成员期间切换会话时，消息可能被发送到新会话的问题。
- 修复搜索关键词变化时短暂显示上一关键词结果，以及部分重复分页静默缺少用户的问题。
- 修复快速变化未读数时，较慢的旧任务栏角标最后覆盖最新值的问题。

## v0.12.0 - 2026-07-15

### 新增

- 快捷搜索同时覆盖会话、跨会话消息、联系人和频道，并支持中文拼音与备注名。
- 通讯录可完整加载 Rocket.Chat 用户目录，成员与文件列表支持服务端分页。
- 从聊天创建 Azure DevOps 工作项时，按项目过程动态加载真实工作项类型。

### 改进

- 优化消息列表滚动锚点、窄窗口布局、线程与侧栏高度、PDF 预览资源回收。
- 完善合并转发和跨房间附件处理，保留可复用内容并阻止原房间受保护文件链接泄露。
- Azure DevOps Bridge 自动识别当前身份，并按当前用户筛选工作项、PR 与构建。
- 统一模态弹窗的 Esc、焦点陷阱和焦点恢复行为。

### 修复

- 修复 Rocket.Chat 文件重定向到外部地址时可能继续携带认证头的问题。
- 修复成员邀请后被较早请求的旧名单覆盖，以及刷新失败时无法可靠重试的问题。
- 修复 Azure DevOps Server 2022 不存在的截止日期字段导致 Bridge 请求失败的问题。
- 修复工作项类型加载失败后仍可使用陈旧类型创建、错误配置被误报为连接成功的问题。
- 修复实时消息更新反复重启快捷搜索，以及创建工作项过程中关闭弹窗可能造成重复提交的问题。
- 修复收藏链接协议校验、弹窗键盘操作、窄视口和多项工作台数据边界。
