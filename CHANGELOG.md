# 更新日志

## v0.29.1 - 2026-07-23

### 新增

- 管家为“找文件、比较 PR、提取承诺、逾期工作项跟进草稿、构建失败关联、周报例行任务、跨重启续跑”七类任务建立结构化 task state；每项任务明确记录能力预检、来源新鲜度、禁止动作和恢复合同（#168、#173）。
- API 与 Codex 现在共享同一套管家 session/engine 生命周期；切换运行模式时会桥接尚未看到的对话，恢复失败时也会把可见 transcript 交给新线程继续（#174）。
- 管家长期记忆收敛为带 `server/account[/project/room]` 作用域的 alias、preference 与 commitment；支持显式召回、撤销、恢复和经批准的旧记忆导入（#176）。
- Today、主动简报、@我 watcher 和例行事务接入同一 workflow runtime，共享任务状态、来源、审批、checkpoint、暂停、失败重试和跨重启恢复（#177）。

### 改进

- “比较这两个 PR”等指代不完整的请求会在进入 API 或 Codex 大脑前只询问缺失的最少信息；用户补充后沿用同一任务，不再猜测对象。
- 工具返回的消息、工作项、PR 和构建来源会同步写入当前任务；完成、停止和失败分别保留可恢复状态，并随独立管家会话跨重启恢复（#173）。
- 所有管家写工具统一执行 typed 参数校验、能力预检和可见审批；首次调用只生成 checkpoint，明确确认后才执行，失败不会被误记为成功（#175）。
- 主动简报最多保留三条值得立即处理的事项，并同时说明原因与建议动作；计划任务在登录前不会误记为已经执行，暂停 Codex 主动任务会真实中断当前回合（#177）。

### 安全与隐私

- 长期记忆不再隐式注入系统提示；召回改为显式工具，写入、撤销、恢复和旧数据导入均需要审批，遗留 v1 记忆默认隔离（#176）。
- 写操作审计只记录 operation、工具、能力、幂等键和状态，不记录参数正文或预览内容；API 与 Codex 不能绕过同一审批边界（#175）。

### 工程

- 保留现有 Provider 与 Codex adapter、Today 结果和 routine/event card 读模型，不新增第二套导航或状态机；主动 workflow session 对人工会话选择器保持隐藏（#174、#177）。
- 新增任务编译、双运行模式切换、typed tool runtime、作用域记忆、主动 workflow、跨账号并发、审批投影与真实 Codex 中断回归（#173–#177）。

### 发布

- 桌面 Release 在当前稳定化阶段仅提供 Windows x64 的 MSI 与 NSIS 安装包、对应 Updater 元数据和签名；macOS 与 Linux 官方安装包延后到平台验证稳定后恢复。
- Windows-only Release 公开但不标记为 Latest，Windows 用户从 Release 页面手动下载；`v0.28.0` 暂时保留为跨平台 Latest，避免现有 macOS/Linux 客户端读取缺少平台条目的更新清单。
